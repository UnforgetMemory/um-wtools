# ADR-0001: Route-Level Code Splitting and Architecture Cleanup

## Status

Accepted

## Context

The project bundles all feature components in a single initial chunk even though
users only interact with one tool at a time. The main bundle included all 6 tool
components plus the full i18n message set on every page load.

Additionally, some modules were found to be:
- `UCard` missing a `#title` slot, causing all card titles to be silently dropped
- `useBase64` acting as a pure pass-through wrapper with no added value
- `markdown.ts` executing module-level side effects via `marked.setOptions()`

## Decision

1. **Route-level code splitting**: Convert all tool route components from static
   imports to dynamic `() => import(...)`. HomeHero remains a static import since
   it is the landing page always loaded first.

2. **UCard #title slot**: Add a named `#title` slot to `UCard` alongside the
   default slot, so consumer components can render card headers.

3. **Remove pass-through composable**: `useBase64` was a 1:1 wrapper of domain
   functions with no added state or lifecycle. Removed it; components import
   domain functions directly. The deletion test confirmed that removing it
   concentrates complexity in the domain layer (where it belongs), not in N
   callers.

4. **Fix module-level side effects**: `marked.setOptions()` moved from module
   scope into a lazy `configureMarked()` call inside the render function, so
   importing the module no longer triggers side effects.

## Consequences

### Positive

- Main bundle reduced from 355 KB to 235 KB (34% reduction)
- Each tool loads on demand, improving initial page load time
- UCard title slots now render correctly across all tools
- markdown.ts is now a pure module (no import-time side effects)
- One less shallow module to maintain

### Negative

- Dynamic imports add a minor async loading delay for tool pages
- The `render()` internal function in markdown.ts uses a callback pattern that
  adds a small indirection

## Compliance

- `lsp_diagnostics` clean on all changed files
- Build passes (exit 0)
- All 90 existing unit tests pass
- No new test failures introduced (37 pre-existing failures unrelated to these
  changes)
